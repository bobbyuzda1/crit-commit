Test Part 1: Test Run

Before launching the real build, let's do a tiny test to make sure your API key works, claude -p runs correctly, and the loop functions. This creates a throwaway test project.

Step 1: Set up a test directory

1

Open a new WSL terminal. Create a temporary test project:

cd /tmp && rm -rf ralph-test && mkdir ralph-test && cd ralph-test && git init

Step 2: Set your API key

2

Replace sk-ant-YOUR-KEY-HERE with your actual key:

export ANTHROPIC_API_KEY=sk-ant-YOUR-KEY-HERE
Where to get your API key: console.anthropic.com/settings/keys. You need billing set up with credits loaded. Never share your key in chat sessions.

Step 3: Verify the API key works

3

Run a single headless Claude call:

claude -p "Say exactly: RALPH TEST SUCCESSFUL" --max-turns 1 --model claude-sonnet-4-20250514
Expected: Output containing "RALPH TEST SUCCESSFUL". If you get an auth error, your API key is wrong or has no credits.

Step 4: Create the test plan

4

Create a tiny 3-task plan. Paste this one command at a time — this is a single-line heredoc that pastes cleanly:

cat > plan.md << 'EOF' # Test Plan - [ ] Task 1: Create a package.json with name "ralph-test" and version "1.0.0" - [ ] Task 2: Create src/hello.js that exports a function hello() returning "Hello from Ralph!" - [ ] Task 3: Create src/hello.test.js that tests the hello function and run it with node to verify EOF

Verify it was created:

cat plan.md

Step 5: Create the test loop script

5

Instead of pasting a loop directly, we create a script file. This avoids all line-break issues:

cat > run-test.sh << 'SCRIPT' #!/usr/bin/env bash set -uo pipefail for i in 1 2 3 4 5; do echo "=== Iteration $i ===" FIRST_LINE=$(head -1 plan.md 2>/dev/null) if [ "$FIRST_LINE" = "STATUS: COMPLETE" ]; then echo "COMPLETE! All tasks done." break fi claude -p "Read plan.md. Find the next unchecked task (- [ ]). Do ONLY that task. Mark it - [x] in plan.md when done. Commit changes. If ALL tasks are checked, add STATUS: COMPLETE as the very first line of plan.md." --allowedTools "Bash,Read,Write,Edit" --max-turns 10 --model claude-sonnet-4-20250514 echo "" DONE=$(grep -c '^\- \[x\]' plan.md 2>/dev/null || echo 0) TOTAL=$(grep -c '^\- \[' plan.md 2>/dev/null || echo 0) echo "Progress: $DONE / $TOTAL tasks" echo "" sleep 2 done SCRIPT

Step 6: Run the test

6

Execute the script. This should complete 3 tasks in about 3-5 minutes:

bash run-test.sh
What to watch for: You should see Claude creating files, running commands, and checking off tasks. Each iteration should show increasing progress (1/3, 2/3, 3/3).

Step 7: Verify test results

7

Create and run the verification script:

cat > verify.sh << 'SCRIPT' #!/usr/bin/env bash echo "=== RALPH TEST VERIFICATION ===" echo "" echo "1. Plan status (line 1):" head -1 plan.md echo "" echo "2. Checked tasks:" grep '^\- \[x\]' plan.md || echo " NONE (problem!)" echo "" echo "3. Unchecked tasks:" grep '^\- \[ \]' plan.md || echo " NONE (all done!)" echo "" echo "4. Files created:" ls -la src/ 2>/dev/null || echo " No src/ directory (problem!)" echo "" echo "5. package.json:" cat package.json 2>/dev/null | head -5 || echo " No package.json (problem!)" echo "" echo "6. Git commits:" git log --oneline 2>/dev/null | head -5 || echo " No commits (problem!)" echo "" echo "7. Test run:" node src/hello.test.js 2>/dev/null || node src/hello.test.mjs 2>/dev/null || echo " Test file not found or failed" echo "" echo "=== END VERIFICATION ===" SCRIPT bash verify.sh
What you should see:
  • Line 1 shows "STATUS: COMPLETE"
  • All 3 tasks checked as [x]
  • No unchecked tasks
  • Files exist in src/
  • package.json with name "ralph-test"
  • 3 git commits
  • Test passes

Step 8: Clean up

8
cd /tmp && rm -rf ralph-test && echo "Cleaned up."
If all checks passed, Ralph is working! Proceed to Part 2.
If something failed: Copy the FULL output of the verification (Step 7) and paste it to me in Claude Code. I'll diagnose the issue.

Build Part 2: Launch Crit Commit Build

Lessons learned from first attempt:
  • The default 15 max turns was not enough for large tasks. The script now uses 40 max turns.
  • Use 60 iterations (not 50) for overnight runs — gives 25 retries beyond the 35 tasks. Unused iterations cost nothing.
  • If you edit ralph-loop.sh while the loop is running, you must stop and restart — the running script uses the old values until restarted.
  • If the loop stalls on partial work, clean up partial files before restarting — ask Claude Code in a separate session to clean the state.

Step 1: Navigate to the project

1
cd /mnt/c/projects/RPG

Step 2: Set your API key

2
export ANTHROPIC_API_KEY=sk-ant-YOUR-KEY-HERE

Step 3: Pre-flight check

3

Create and run the pre-flight check:

cat > preflight.sh << 'SCRIPT' #!/usr/bin/env bash echo "=== Pre-flight Check ===" echo "plan.md: $(test -f plan.md && echo 'OK' || echo 'MISSING!')" echo "ralph-loop.sh: $(test -f ralph-loop.sh && echo 'OK' || echo 'MISSING!')" echo "CLAUDE.md: $(test -f CLAUDE.md && echo 'OK' || echo 'MISSING!')" echo "Detailed plan: $(test -f docs/superpowers/plans/2026-03-15-crit-commit-implementation.md && echo 'OK' || echo 'MISSING!')" echo "Design spec: $(test -f docs/superpowers/specs/2026-03-15-crit-commit-design.md && echo 'OK' || echo 'MISSING!')" echo "API key set: $(test -n "${ANTHROPIC_API_KEY:-}" && echo 'OK' || echo 'NOT SET!')" echo "Claude CLI: $(which claude 2>/dev/null || echo 'NOT FOUND!')" echo "Tasks in plan: $(grep -c '^\- \[ \]' plan.md 2>/dev/null || echo 0)" echo "===========================" SCRIPT bash preflight.sh
All should say OK. Tasks should be 35. Fix anything that says MISSING or NOT SET before proceeding.

Step 4: Launch the overnight build

4

This is it. This runs for several hours autonomously:

bash ralph-loop.sh 60
What happens now:
  • Loop reads plan.md, implements one task per iteration
  • 40 max turns per iteration (enough for the largest tasks)
  • 60 max iterations (35 tasks + 25 buffer for retries)
  • Most tasks use Sonnet. Tasks 12, 13, 20, 27, 32, 33 auto-use Opus
  • If a task fails 3x on Sonnet, it auto-escalates to Opus
  • Each iteration takes 1-5 minutes
  • 35 tasks = roughly 3-6 hours total
  • Logs to .ralph/ralph-*.log
  • Stops automatically when complete — unused iterations cost nothing
Keep this terminal open! If you close it, the build stops. Use tmux (see below) to detach safely.

Optional: Run in tmux (close terminal safely)

*

Start tmux first, then launch inside it:

tmux new -s ralph

Then inside tmux:

cd /mnt/c/projects/RPG && export ANTHROPIC_API_KEY=sk-ant-YOUR-KEY-HERE && bash ralph-loop.sh 60

To detach (terminal keeps running): press Ctrl+B then D

To reconnect later:

tmux attach -t ralph

Troubleshooting

Task keeps failing / "Reached max turns"

!

If the same task fails repeatedly, it means the task needs more turns than allowed. Stop the loop, increase max turns, clean up partial work, and restart:

touch /mnt/c/projects/RPG/.ralph/KILLSWITCH

Wait for the loop to stop in your terminal. Then increase max turns by editing ralph-loop.sh — find MAX_TURNS= and increase the number. Then clean partial files, remove killswitch, and relaunch:

rm /mnt/c/projects/RPG/.ralph/KILLSWITCH
bash ralph-loop.sh 50
Important: If you edit ralph-loop.sh while it's running, the changes do NOT take effect until you stop and restart the script. The running bash process uses the values it loaded at startup.

Loop won't stop after killswitch

!

The killswitch is checked between iterations, not during. The current claude -p call must finish first (up to 60 seconds). If it's truly stuck, press Ctrl+C in the terminal running the loop.

Need to restart from scratch after partial failures

!

If early tasks left partial files that confuse later attempts, ask Claude Code (in a separate interactive session) to clean up the partial state. Then reset the checkboxes in plan.md for the failed tasks and relaunch.

API key or billing error

!

If you see auth errors in the log, your key may have expired or run out of credits. Check console.anthropic.com/settings/billing. Set the new key and relaunch — the loop picks up from the last unchecked task.
